# Держимся

Эмоциональный компаньон для офисных работников. Помогает честно понимать своё состояние, замечать выгорание и ориентироваться на рынке труда — тоном понимающего коллеги, с самоиронией, без токсичной продуктивности.

Язык интерфейса — **русский**.

## Стек и платформа

- **React 19 + TypeScript + Vite** (web), **Node 24**. Линт — oxlint, формат — Prettier. Конвенции и тулчейн — [docs/conventions.md](docs/conventions.md).
- **Первая цель — Telegram Mini App.** Архитектура строится так, чтобы платформенные зависимости (в первую очередь авторизация) были спрятаны за провайдерами и легко заменялись для PWA / сайта / WebView. См. [docs/architecture.md](docs/architecture.md).
- **Frontend-first.** Сначала описали клиентские типы и моканые API-ручки на фронте; теперь поднимаем реальный бэкенд (Node + TypeScript, **Fastify + Prisma + PostgreSQL**) в папке `backend/`, общие API-типы — в `contracts/`. Бэкенд и контракты — на одном уровне с `src/`. См. [docs/backend.md](docs/backend.md).

## Конвенции

- UI собираем строго из дизайн-системы: компоненты из `~/shared/ui` и токены из `src/shared/styles` (описание — [docs/design-system.md](docs/design-system.md)), сверяясь с макетами экранов в `docs/design/`.
- Tone of voice копирайта — по [docs/product.md](docs/product.md): спокойно, честно, самоиронично, без давления и осуждения.
- Платформенные API (Telegram, авторизация, хранилище, сеть) — только через провайдеры/абстракции, никогда напрямую из компонентов.

## Команды

> Node 24 (`.nvmrc`). Перед командами — `nvm use`.

- `yarn dev` — дев-сервер Vite (https через mkcert)
- `yarn build` — типчек + прод-сборка
- `yarn lint` — oxlint
- `yarn format` — Prettier (`format:check` — проверка без записи)
